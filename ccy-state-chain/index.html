Framework · Eight-State Reference

The CCY State Chain

The CCY State Chain maps the eight institutional states through which currency passes inside enterprise finance — from quoted market price to audit-defensible monetary truth.

State Chain Doctrine

A currency does not have one meaning across enterprise finance.
It has a state, a role, a custody position, and a consequence.

The CCY State Chain · AccountCcy.com

Why The State Chain Matters

The same monetary event can pass through multiple currency states before it becomes defensible.

A financial event may begin as a quoted rate, become a transaction obligation, settle as cash movement, enter an account, be maintained in a ledger, translate into a reporting currency, consolidate at group level, and finally face audit scrutiny.

Each state carries a different question. Each transition can create distortion if it is not governed. The CCY State Chain makes these states visible so they can be named, linked, controlled, and reviewed.

State 04 — Account Currency — is the critical control point because it is where monetary movement enters accounting custody and begins to affect the ledger, reporting, consolidation, and audit layers.

Eight Custody States

From quoted price to audit reality.

The state chain gives AccountCcy.com its navigable architecture. Every glossary term, risk page, comparison page, ERP page, CFO guide, and diagnostic tool should map back to one or more states.

State 01

Quoted Currency

Currency as a market price before a transaction, obligation, account, ledger, report, or audit trail exists.

State 02

Transaction Currency

The currency in which the financial event is denominated and the obligation takes its first formal identity.

State 03

Settlement Currency

The currency in which cash actually transfers, creating consequences when settlement diverges from transaction currency.

State 04 · Control Point

Account Currency

The currency assigned to an account for recording, maintaining, reconciling, and controlling balances inside the accounting system.

State 05

Ledger Currency

The currency in which entity books and balances are maintained, commonly aligned with the functional currency depending on framework and system configuration.

State 06

Reporting Currency

The currency in which results are presented for management, regulatory, investor, or internal reporting purposes.

State 07

Consolidation Currency

The group-level currency into which multiple entity ledgers, reporting layers, and adjustments are unified.

State 08

Audit Reality

The final defensible state where monetary figures can be traced, supported, reconciled, reviewed, and defended.

State 04

Account Currency is the custody gate.

The CCY State Chain gives State 04 special weight because account currency is where external monetary movement becomes internal accounting record. Weakness at this point is not isolated; it travels downstream.

State Failure

Confusing states creates hidden accounting distortion.

When transaction currency, settlement currency, account currency, ledger currency, and reporting currency are treated as interchangeable, the organization loses the ability to trace where monetary truth changed state and where distortion entered the system.
