# AccountCcy.com Framework Versioning

## Document Status

This document defines how AccountCcy.com versions its framework, terminology, diagnostic logic, and governed reference architecture.

Versioning makes the asset harder to imitate because it gives the methodology institutional memory.

---

## 1. Current Framework Version

Current framework version: `v1.0`

Version name: `Currency State Control Framework v1.0`

Core architecture:

- Chain of Custody for Monetary Truth
- Currency State Control Framework
- CCY State Chain
- Account Currency as State 04 control point
- Audit Reality as State 08 terminal evidence posture
- Currency State Diagnostic as local-first maturity signal

---

## 2. Version Scope

Framework versioning applies to:

- governing thesis
- state definitions
- state sequence
- glossary authority map
- diagnostic maturity logic
- evidence maps
- control artifacts
- claim boundaries
- source boundaries
- public methodology

It does not apply to routine copy edits unless they alter meaning.

---

## 3. Version Change Types

### Patch Change

Examples:

- wording clarification
- typo correction
- non-semantic link correction
- metadata adjustment

Decision log entry:

Optional unless trust, SEO, or governance is affected.

### Minor Change

Examples:

- new evidence artifact class
- diagnostic wording update
- new reference page inside existing architecture
- glossary expansion inside current framework
- source or claim boundary refinement

Decision log entry:

Required.

### Major Change

Examples:

- changing the eight-state sequence
- redefining Account Currency as State 04
- changing the governing thesis
- changing the diagnostic maturity model
- replacing the framework hierarchy
- changing acquisition posture materially

Decision log entry:

Required before publication.

Major change should also update this file and `PUBLIC_CHANGELOG.md`.

---

## 4. Version Integrity Rules

The state chain must not be changed casually.

The current eight-state sequence is:

1. Quoted Currency
2. Transaction Currency
3. Settlement Currency
4. Account Currency
5. Ledger Currency
6. Reporting Currency
7. Consolidation Currency
8. Audit Reality

Any change to this sequence is a major framework change.

---

## 5. Diagnostic Versioning

The diagnostic should record:

- question set version
- maturity level model
- state weighting logic
- report interpretation boundary
- non-advisory disclaimer

Current diagnostic version: `v1.0`

Current maturity levels:

- Fragmented
- Configured
- Governed
- Controlled
- Audit-Defensible

---

## Final Versioning Rule

If a change would affect how a professional interprets the framework, it must be versioned and logged.
