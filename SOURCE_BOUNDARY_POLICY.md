# AccountCcy.com Source Boundary Policy

## Document Status

This document defines how AccountCcy.com separates original framework language from external terminology, professional practice, standards references, vendor behavior, and public evidence.

The policy protects trust.

It prevents AccountCcy.com from presenting original concepts as official standards or unsupported interpretations as fact.

---

## 1. Source Boundary Principle

AccountCcy.com may build original conceptual architecture.

It must distinguish that architecture from:

- accounting standards
- audit standards
- ERP vendor documentation
- treasury practice
- financial reporting practice
- regulatory requirements
- professional advice
- external market evidence

The framework can be original. Claims about the world must be bounded.

---

## 2. Source Classes

### Class A: AccountCcy Original Framework

Examples:

- Chain of Custody for Monetary Truth
- Currency State Control Framework
- CCY State Chain
- Audit Reality as a terminal currency-state concept
- account currency control as a custody discipline

Use:

Original AccountCcy methodology and reference architecture.

Boundary:

Never present as official accounting standards or regulatory rules.

### Class B: Established Professional Terminology

Examples:

- functional currency
- transaction currency
- reporting currency
- consolidation currency
- currency translation
- FX revaluation
- general ledger
- subledger

Use:

Reference and explanation.

Boundary:

Avoid overclaiming. Where standards, jurisdiction, or vendor behavior matter, use restrained language and refer users to qualified professionals.

### Class C: ERP and System Behavior

Examples:

- account master configuration
- currency master data
- posting logic
- revaluation runs
- reporting extracts

Use:

System behavior framing.

Boundary:

No vendor affiliation. No copied vendor instructions. No implementation advice.

### Class D: External Evidence

Examples:

- backlinks
- citations
- professional mentions
- tool usage signals
- qualified inquiries
- external references

Use:

Evidence of public trust and market recognition.

Boundary:

Record only verified evidence. Do not inflate weak mentions into authority claims.

---

## 3. Vendor Boundary

AccountCcy.com may discuss ERP categories and general system behavior.

It must not imply endorsement, partnership, certification, or affiliation with any ERP vendor unless such relationship is formally documented.

Vendor names, if used later, must be contextual and restrained.

---

## 4. Standards Boundary

AccountCcy.com may explain how currency-state custody relates to professional accounting concepts.

It must not:

- determine functional currency for a user
- interpret standards for a specific entity
- provide audit conclusions
- provide tax or legal conclusions
- tell a user how to configure an ERP system

---

## 5. Citation Boundary

Public citations should support factual or terminology claims.

Citations should not be used to create false authority for AccountCcy original concepts.

Original framework concepts should be labeled as original framework concepts.

---

## Final Source Boundary Rule

When a claim could be mistaken for professional authority, narrow it, qualify it, or do not publish it.
