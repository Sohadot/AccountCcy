# AccountCcy.com Asset Intelligence Factory Plan

## Document Status

This document converts AccountCcy.com from a reference site into a governed category intelligence asset.

It does not replace `FOUNDATION_DOCTRINE.md`, `ASSET_THESIS.md`, `ARCHITECTURE.md`, `CONTENT_GOVERNANCE.md`, `SEO_POLICY.md`, `MONETIZATION_PRINCIPLES.md`, or `QUALITY_GATE.md`.

It operationalizes them.

The governing thesis remains:

**AccountCcy.com is the chain of custody for monetary truth inside enterprise finance.**

---

## 1. Domain Thesis

AccountCcy.com owns the category question:

**How does currency become defensible accounting truth inside enterprise finance?**

The asset must never reduce account currency to a setup field, ISO code, dictionary entry, or keyword opportunity.

Account currency is treated as the control point where monetary movement enters accounting custody.

---

## 2. Category Language

The asset should consistently use and reinforce controlled vocabulary:

- monetary truth
- chain of custody
- currency state
- CCY State Chain
- account currency control
- ledger measurement
- reporting visibility
- consolidation assembly
- audit reality
- evidence trail
- monetary reconstruction
- configuration drift
- assurance debt

New pages must strengthen this language rather than invent unrelated slogans.

---

## 3. Ontology

The AccountCcy ontology is organized around eight currency states:

1. Quoted Currency
2. Transaction Currency
3. Settlement Currency
4. Account Currency
5. Ledger Currency
6. Reporting Currency
7. Consolidation Currency
8. Audit Reality

Each state must answer:

- What monetary identity does this state govern?
- Who owns the state?
- Which artifacts prove it?
- What downstream state receives its output?
- What breaks if the state is confused with another?

---

## 4. Standard

The AccountCcy standard is not an accounting standard and must not be presented as one.

It is a reference-grade control standard for evaluating whether currency-state custody is:

- defined
- owned
- configured
- evidenced
- reviewed
- reproducible
- audit-defensible

Any public claim must distinguish original AccountCcy framework language from professional standards, vendor documentation, or regulated advice.

---

## 5. Protocol

Every diagnostic, checklist, framework page, or premium brief must follow this protocol:

1. identify the relevant currency state
2. define the custody question
3. identify the evidence artifact
4. name the control owner
5. describe the failure mode
6. connect the result to framework and glossary nodes
7. state the non-advisory boundary

If a page cannot pass this protocol, it remains draft.

---

## 6. Engine / Tool

The first public engine is the `Currency State Diagnostic`.

It must remain:

- local-first
- transparent
- non-submitting
- non-certifying
- educational
- connected to the CCY State Chain
- useful enough to produce a maturity signal

Future tools may include:

- Account Currency Control Checklist
- ERP Currency Configuration Risk Scanner
- FX Revaluation Readiness Check
- Audit Reality Traceability Checklist

No tool may imply automated professional judgment.

---

## 7. Reference Layer

The reference layer is the defensibility moat.

Priority pages must deepen:

- Account Currency definition
- State 04 Account Currency
- CCY State Chain
- Currency State Control Framework
- Monetary Truth Chain of Custody
- CFO Guide
- Glossary authority clusters
- ERP ledger control pages
- audit evidence and close discipline pages

No page may exist only because a keyword exists.

---

## 8. Governance

Governance is stricter than growth.

Publication requires:

- no placeholders
- no unsupported professional claims
- no broken internal links
- no draft links
- no domain-sale framing
- no low-trust monetization
- no vendor affiliation confusion
- no advice language
- clear canonical URL
- clear relation to the governing thesis

Major changes should be traceable through this file, `QUALITY_GATE.md`, and `DECISION_LOG.md`.

---

## 9. Interface Thesis

The interface must embody the asset thesis.

It should feel like a custody system:

- ordered
- quiet
- precise
- evidentiary
- institutional
- structured around state transitions

Visual design must clarify the system rather than decorate it.

WebGL, 3D, heavy animation, or decorative spectacle should not be used unless it explains custody transitions better than static structure.

---

## 10. Monetization

Revenue is allowed only where it strengthens authority.

Preferred revenue layers:

- diagnostic reports
- premium briefs
- framework licensing
- professional training packs
- audit intake education
- reference sponsorships with strict independence
- governed taxonomy licensing

Rejected revenue layers:

- display ads
- generic affiliate offers
- retail forex leads
- crypto promotions
- urgency tactics
- cheap templates
- public domain-sale pressure

The first monetization candidate is a governed diagnostic report derived from the Currency State Diagnostic.

---

## 11. Buyer Logic

The strategic buyer is not buying only traffic or code.

The buyer is acquiring:

- domain fit
- category language
- controlled ontology
- reference architecture
- internal link graph
- diagnostic surface
- professional positioning
- acquisition-compatible monetization path
- governance discipline

The acquisition logic strengthens when the asset becomes difficult to replicate as a whole.

A buyer may reproduce a tool, but reproducing the name, thesis, vocabulary, reference stack, governance posture, and accumulated authority is materially harder.

---

## 12. Immediate Execution Sequence

1. Publish the Currency State Diagnostic as a local-first maturity engine.
2. Publish a restrained Strategic Acquisition Posture page.
3. Keep all weaker tools in draft until content and link integrity are complete.
4. Expand the reference layer through state-specific and evidence-specific pages only where the quality gate is met.
5. Use the diagnostic report as the first dignified monetization layer.
6. Preserve the asset as reference infrastructure before commercial surface.

---

## Final Rule

AccountCcy.com should not look like a site trying to sell a domain.

It should look like the category already has a reference system, and that AccountCcy.com is where the system lives.
