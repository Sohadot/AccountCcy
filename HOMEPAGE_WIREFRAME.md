# AccountCcy.com Homepage Wireframe

## Document Status

This document defines the homepage wireframe for AccountCcy.com.

It translates the asset doctrine, design system, UI components, and CSS foundation into a page-level experience.

This is not a visual mockup.

This is the strategic structure that governs the homepage before `index.html` is written.

The homepage must establish AccountCcy.com as a sovereign-grade reference asset, not as a generic finance site, currency tool, SaaS landing page, or domain sales page.

---

## Governing Homepage Thesis

**AccountCcy.com is the chain of custody for monetary truth inside enterprise finance.**

The homepage must make this thesis visible, understandable, and inevitable within the first screen.

The visitor should understand quickly that AccountCcy.com is about the controlled path through which currency moves from market price to accounting truth.

The homepage must communicate:

- monetary truth
- chain of custody
- currency state control
- account currency as the control point
- enterprise finance seriousness
- reference-grade authority
- future diagnostic and acquisition logic

---

## Primary UX Objective

Within the first ten seconds, a qualified visitor should understand:

1. This is not a currency converter.
2. This is not a forex site.
3. This is not a domain-for-sale page.

6. Account currency is the control point inside the chain.

8. The subject matters to CFOs, controllers, ERP architects, financial close teams, treasury systems, and audit-facing finance professionals.

# 3. Problem Section — Monetary Truth Fragmentation

## Purpose

This section defines the problem AccountCcy.com owns.

It should make clear that the problem is not currency volatility alone.

The deeper problem is fragmentation of monetary truth across enterprise finance systems.

## Section Eyebrow

**The Hidden Problem**

## Section Title

**Enterprise finance does not suffer only from currency volatility. It suffers from monetary truth fragmentation.**

## Core Copy

A financial event can carry multiple currency identities before it becomes defensible reporting reality.

It may be quoted in one currency, transacted in another, settled in another, recorded in an account, maintained in a ledger, translated for reporting, consolidated at group level, and reviewed under audit.

When these states are misunderstood or weakly governed, the result can be reconciliation complexity, FX distortion, reporting inconsistency, close-process burden, and audit exposure.

## Supporting Bullets

- The same monetary event can hold multiple currency identities.
- Treasury may see exposure before accounting sees distortion.
- ERP configuration may become a financial control issue.
- Reporting truth depends on the chain, not the final number alone.
- Account currency is where external movement enters accounting custody.

## Component

Use:

**Problem Ledger Panel**

A two-column panel:

Left: problem statement.  
Right: risk list.

Risks:

- reconciliation complexity
- FX distortion
- reporting inconsistency
- revaluation issues
- translation effects
- consolidation confusion
- audit exposure

## UX Rule

The problem must feel real, not invented.

Language must remain precise and defensible.

# 5. Control Point Section — Account Currency

## Purpose

This section explains why the domain name matters.

The site is called AccountCcy.com because account currency is the control point in the chain.

## Section Eyebrow

**The Control Point**

## Section Title

**Account currency is where monetary movement enters accounting custody.**

## Core Copy

Before the account currency state, currency belongs primarily to market, transaction, and settlement behavior.

After the account currency state, it begins to propagate into ledger, reporting, consolidation, and audit layers.

This is why account currency must not be treated as a minor configuration field.

It is a financial control discipline.

## Definition Box

**Account currency** is the currency assigned to an account for recording, maintaining, reconciling, and controlling financial balances inside an accounting or ERP system.

## Consequence Panel

A weak account currency layer can create downstream distortion.

A strong account currency layer can preserve downstream clarity.
