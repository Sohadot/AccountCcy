# AccountCcy.com Methodology

## Document Status

This document defines the methodology used by AccountCcy.com to create, maintain, and review its framework, terminology, diagnostic logic, and reference content.

This is not an accounting standard.

This is not professional advice.

This is an original reference methodology for organizing currency-state custody inside enterprise finance.

The governing thesis remains:

**AccountCcy.com is the chain of custody for monetary truth inside enterprise finance.**

---

## 1. Methodological Purpose

The purpose of the AccountCcy methodology is to make currency-state transitions explicit.

The methodology asks:

- What currency identity is present?
- Which state owns that identity?
- Which artifact proves it?
- Which system records it?
- Which control owner reviews it?
- Which downstream state inherits it?
- What breaks if the state is confused with another?

The methodology exists to convert ambiguous currency language into governed reference structure.

---

## 2. Core Method

Every AccountCcy reference element should follow this sequence:

1. Define the monetary state.
2. Locate the state inside the CCY State Chain.
3. Identify neighboring states that are often confused with it.
4. Identify evidence artifacts that make the state reviewable.
5. Identify control owners or control forums.
6. Identify failure modes.
7. Link the concept back to framework, glossary, diagnostic, and governance documents.
8. State the non-advisory boundary.

If an element cannot pass this sequence, it should remain draft or internal.

---

## 3. Original Framework Boundary

AccountCcy.com uses original framework language, including:

- Chain of Custody for Monetary Truth
- Currency State Control Framework
- CCY State Chain
- Audit Reality
- account currency control as a custody discipline

These are AccountCcy framework concepts.

They must not be described as official accounting standards, regulatory requirements, vendor instructions, or audit procedures.

The methodology may use established terms such as functional currency, transaction currency, reporting currency, currency translation, FX revaluation, and general ledger. Those terms must be handled with source discipline and claim restraint.

---

## 4. Evidence Method

Each currency-state page should identify evidence classes.

Evidence classes may include:

- policy matrices
- account master extracts
- transaction extracts
- bank confirmations
- revaluation logs
- reporting bridges
- consolidation packages
- audit trail indexes
- exception closure logs

Evidence classes are not universal requirements. They are reference categories for structured review.

The methodology does not say that every organization must use the same artifacts. It says that monetary truth becomes stronger when currency-state claims can be replayed through evidence.

---

## 5. Diagnostic Method

The Currency State Diagnostic is a reference maturity tool.

It is designed to produce a local maturity signal across the CCY State Chain.

It does not:

- certify compliance
- replace professional judgment
- provide accounting advice
- provide audit advice
- provide tax advice
- provide legal advice
- provide ERP implementation advice
- collect submitted answers by default

Diagnostic questions should map to states, weights, evidence posture, and governance maturity.

---

## 6. Content Method

AccountCcy content should be created only when it strengthens at least one of these:

- the governing thesis
- CCY State Chain clarity
- account currency control
- ERP currency configuration discipline
- financial close evidence
- reporting integrity
- audit defensibility
- diagnostic usefulness
- strategic buyer legibility

No page should exist only because a keyword exists.

---
